# AI 知识库系统 — 项目愿景

> **AI 信息洪流中的智能过滤器，多 Agent 协作的内容工厂**

---

## 1. 愿景

**让每一个 AI 从业者每天早上打开一个页面，就能看完昨天全世界最重要的 AI 进展。**

不再需要刷 5 个信息源、读 20 篇论文摘要、翻 100 条 HN 评论。系统在背后完成采集→分析→整理→发布全流程，你只需要消费最终产物。

---

## 2. 问题

| 痛点 | 描述 |
|------|------|
| **信息过载** | GitHub Trending 每天数百 repo，HN 日均 150+ 帖，arXiv 日更 300+ AI 论文 |
| **噪音远大于信号** | 真正值得关注的占比不到 5%，但每条都要人工点开判断 |
| **跨源关联缺失** | 一篇 arXiv 论文 → 对应 GitHub 实现 → HN 讨论，三者分散在不同平台 |
| **重复劳动** | 资深 AI 工程师每天花 30-60 分钟做这件事，且是纯体力活 |

---

## 3. 解决方案

多 Agent 协作流水线，端到端自动化：

```
[采集层] → [分析层] → [整理层] → [发布层]
   ↓            ↓            ↓            ↓
  Crawler     Analyzer    Summarizer    Publisher
  Agents      Agents      Agents        Agents
```

每个环节由专门 Agent 执行，Agent 之间通过结构化数据协议协作。

### 3.1 采集层 — Collector Agents

- **GitHub Agent**：抓取 Trending（每日/每周/语言过滤），提取 repo 名、描述、Star 数、最近 commit
- **HN Agent**：抓取 Front Page + /newest，提取标题、分数、评论数、首条评论摘要
- **arXiv Agent**：抓取 cs.AI / cs.LG / cs.CL 新论文，提取标题、作者、摘要、关键词
- **去重 Agent**：跨源去重（同一篇论文同时出现在 arXiv 和 HN 时合并）

### 3.2 分析层 — Analyzer Agents

- **热度评分 Agent**：综合 Star 增速、HN 分数、论文引用潜力，给每项计算一个 **关注度指数** (0-10)
- **分类打标 Agent**：自动打标签（LLM / 多模态 / Agent / 推理 / 工程 / 应用...）
- **关联发现 Agent**：检测"arXiv 论文 X + GitHub 实现 Y + HN 讨论 Z"属于同一主题，自动合并
- **质量过滤 Agent**：过滤低质量内容（如空壳 repo、营销帖），设定最低阈值

### 3.3 整理层 — Summarizer Agents

- **每日简报 Agent**：生成今日 Top 10，每条含 1-2 句摘要 + 原始链接
- **深度解读 Agent**：为 Top 3 项目写 200-300 字分析（解决了什么问题、技术方案、值得关注的理由）
- **趋势观察 Agent**：识别本周/本月 recurring 主题（如"所有人都在做 MCP Server"、"Agent 评估框架井喷"）
- **个性化 Agent**：根据订阅标签调整内容排序和权重

### 3.4 发布层 — Publisher Agents

- **排版 Agent**：将内容渲染为最终格式（Web / Newsletter / WeChat / Twitter）
- **发布 Agent**：自动部署到 Web 站点 / 发送邮件 / 推送社交账号
- **效果追踪 Agent**：记录各渠道阅读量、点击率，反馈给分析层做排序优化

## 4. 系统架构

```
┌──────────────────────────────────────────────┐
│                  Web UI / API                  │
├──────────────────────────────────────────────┤
│                                                │
│  ┌─────────┐  ┌─────────┐  ┌──────────────┐  │
│  │ 采集     │  │ 分析     │  │ 整理→发布     │  │
│  │ Worker   │→│ Worker   │→│ Worker       │  │
│  │ Pool     │  │ Pool     │  │ Pool         │  │
│  └────┬────┘  └────┬────┘  └──────┬───────┘  │
│       │             │              │           │
│  ┌────▼─────────────▼──────────────▼───────┐  │
│  │           Message Queue (Redis)          │  │
│  └────────────────┬─────────────────────────┘  │
│                   │                            │
│  ┌────────────────▼─────────────────────────┐  │
│  │          持久化层 (PostgreSQL + S3)       │  │
│  └──────────────────────────────────────────┘  │
│                                                │
│  ┌──────────────────────────────────────────┐  │
│  │  LLM 网关 (统一 API 层, 多模型路由)       │  │
│  └──────────────────────────────────────────┘  │
└──────────────────────────────────────────────┘
```

### 设计原则

- **Agent 间松耦合**：通过 Message Queue 通信，任一 Agent 可单独升级/替换
- **LLM 无关**：通过 LLM 网关抽象，可切换 Claude / GPT / 开源模型
- **增量处理**：增量爬取 + 增量分析，避免重复计算
- **人工 in the loop**：自动产出的内容需人工审核确认后才发布（可配置为自动发布）
- **可观测**：每个 Agent 输出结构化日志，可追溯"这条内容是怎么来的"

## 5. 用户画像

| 角色 | 使用场景 | 核心诉求 |
|------|----------|----------|
| **AI 研究者** | 每天早上看 arXiv 精选 | 不错过重要论文，跨源关联 |
| **AI 工程师** | 关注实用项目和技术动态 | GitHub 热度+HN 讨论, 快速评估 |
| **AI 创业者** | 追踪赛道趋势和竞品动态 | 趋势判断, 投资信号 |
| **AI 博主/编辑** | 获取选题素材 | 深度素材, 跨源整合 |

---

## 6. 成功指标

| 指标 | 目标 | 衡量方式 |
|------|------|----------|
| 每日覆盖 | 覆盖 90%+ 当日高关注内容 | 人工抽样验证 |
| 准确率 | 分类打标准确率 > 90% | 每周抽样标注 |
| 时效性 | 内容发布延迟 < 2 小时（从源发布算起） | 系统监控 |
| 用户阅读时长 | 日均阅读 > 5 条 | 产品分析 |
| 漏报率 | 高关注内容漏报率 < 5% | 用户反馈 + 周度复盘 |

## 7. 路线图

### Phase 1 — 基础管线（MVP）
- GitHub Trending 采集 + 热度排序
- arXiv 论文采集 + 摘要提取
- 每日 Top 10 简报生成（Web 页面展示）
- 人工审核发布流程

### Phase 2 — 智能增强
- HN 采集 + 跨源关联
- 深度解读 Agent（Top 3 项目分析）
- 分类导航 + 搜索
- 邮件订阅推送

### Phase 3 — 个性化与生态
- 个性化订阅标签
- 趋势发现 Agent（周报/月报）
- Twitter / WeChat 自动推送
- RSS 输出 / API 开放

### Phase 4 — 网络效应
- 用户反馈回路（顶/踩影响排序）
- 社区贡献（用户可投稿/纠偏）
- Agent 插件市场（第三方可写自定义分析 Agent）
- 开源核心 + 托管商业版

## 8. 竞品对照

| 竞品 | 我们的差异点 |
|------|-------------|
| Feedly / Inoreader | 纯 RSS，无分析理解层，无跨源关联 |
| Paper Digest | 仅论文，无 GitHub/HN，AI 总结质量一般 |
| GitHub Trending 官方 | 无分类无分析，每日刷新无历史积累 |
| TLDR Newsletter | 人工为主，规模受限，无个人化 |
| 同类 Agent 项目 | 多为单人玩具，缺乏工程化管线 |

---

## 9. 核心理念

> **AI Agent 不是为了取代人类判断，而是为了把人类从"筛选"中解放出来，把精力留给"判断"。**

这个系统的目标是：**把无聊的事情交给 Agent，把有趣的事情留给人类。**
